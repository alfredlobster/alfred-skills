# Escalation And Next Skills

Choose the next skill based on the main blocker after the plan is constructed.

## Prefer experiment-designer

Use when:
- the stage plan reveals that the discriminating experiment is still poorly specified
- controls, readouts, or feasibility assumptions remain too vague for execution
- the plan is blocked by weak test design rather than sequencing

## Prefer critic-reviewer

Use when:
- the plan is fragile because important risks, confounders, or hidden assumptions remain
- the selected path should be stress-tested before committing further resources
- stage gates depend on reasoning that still looks weak or biased

## Prefer ranking-and-decision

Use when:
- the plan still depends on unresolved option comparison
- multiple live branches remain credible and expensive enough to warrant explicit ranking
- the chosen path is clearly provisional pending one or two decision criteria

## Prefer research-intake-brief

Use when:
- execution planning exposed a reframed problem or a materially different objective
- the original problem statement is no longer the right anchor
- major missing context means the work should be reframed before proceeding

## General rule

Prefer the closest upstream corrective skill that would reduce the plan's biggest fragility.
If the plan is ready and the user mainly needs operational execution outside the co-scientist chain, keep `recommended_next_skill` conservative and explain the next action clearly.
