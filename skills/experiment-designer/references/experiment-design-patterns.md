# Experiment Design Patterns

## Compact mode

Use when the user wants a fast draft.

Include:
- one to three experiments maximum
- minimal but explicit controls
- short feasibility notes
- short open-question list
- concise YAML handoff

Do not over-specify protocol mechanics.

## Default mode

Use in normal situations.

Include:
- two to five experiments
- sequencing and prioritization rationale
- controls, readouts, confounders, and feasibility for each experiment
- a clear decision path

## Rigorous mode

Use when stakes or ambiguity are high.

Include:
- explicit alternative hypotheses
- staged exploratory versus confirmatory logic
- stronger treatment of reproducibility, measurement validity, and operational risk
- explicit failure modes and interpretation pitfalls
- clearer stop or pivot criteria

## Cross-mode guidance

Across all modes:
- optimize for discriminating power, not volume
- prefer interpretable readouts
- expose assumptions rather than hiding them
- state what result would change the next decision
