# RUP Transformation Rules

## Actor catalog
- one entry per actor ID
- preserve original name and description
- add classification readiness note if complexity is not yet obvious

## Use-case specification
- preserve use-case IDs
- map main success scenario into main flow
- map alternate and exception flows separately
- include preconditions and postconditions explicitly
- keep transaction count empty until deterministic counting step runs

## Supplementary specification
- collect all NFRs by ID
- preserve category and priority
- keep unresolved or ambiguous NFRs visible
